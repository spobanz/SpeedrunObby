# 14-Day Blueprint: Speedrun Obby (Checkpoints + Coins)

## Week 1 — Ship the Playable Skeleton

### Day 1 — Setup + Rules
- Build the folder/part structure in Studio
- Add Start, Finish, 3 checkpoints, 10 coins
- Goal: touching parts drives the loop

### Day 2 — Timer + Checkpoint Respawn
- Verify timer updates every frame
- Checkpoint touch saves respawn location
- Goal: reliable speedrun feel

### Day 3 — Finish Logic + Best Time
- Finish pad ends run and restarts timer
- Update player best time if improved
- Goal: complete run loop works consistently

### Day 4 — Save Data
- Save/load coins + best time with DataStore
- Test Studio API services enabled
- Goal: progression persists

### Day 5 — Map Iteration
- Expand course to ~45 seconds average run
- Add 5–10 checkpoints total
- Goal: map feels fair and readable

### Day 6 — Juice + Clarity
- Add coin pickup sound
- Add clear checkpoint and finish signage
- Goal: polish without feature creep

### Day 7 — Playtest + Fix
- Run 3–5 player tests
- Fix softlocks and unclear jumps
- Goal: stable Version 1

## Week 2 — Replayability (Still Lightweight)

### Day 8 — Leaderboard
- Surface server best-time and coin ranking
- Goal: immediate competition

### Day 9 — Daily/Weekly Goal
- Add one light objective (example: collect 10 coins)
- Goal: retention hook

### Day 10 — Cosmetics
- Add one coin-based cosmetic toggle (trail/aura)
- Goal: reward loop depth

### Day 11 — Monetization Hooks (Optional)
- Add one gamepass hook (double coins/speed/VIP trail)
- Goal: monetization-ready architecture

### Day 12 — Store Assets
- Ship thumbnail, title, description
- Goal: discovery-ready listing

### Day 13 — Final QA
- Reset test, data wipe test, mobile test
- Goal: no blocker bugs

### Day 14 — Publish + Next Steps
- Publish Version 1
- Write next three 1–2 hour updates
- Goal: keep shipping cadence
